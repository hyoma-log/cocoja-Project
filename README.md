#  ココじゃ

## ■ サービス概要
**「ガイドブックには載っていない、地域の魅力をみんなでつくる」**

旅行の口コミサイトはたくさんありますが、このSNSは少し違います。 「地元の人が通う最高の定食屋」や「名もなき絶景」など、あなたの独自の視点で都道府県の魅力を発信してください。
投稿が集まると独自のランキングに反映されるため、ユーザー同士で楽しみながら応援することができます。「次はここに行ってみたい！」という発見や、地域の人との交流が生まれる、愛着と共感にあふれた新しいコミュニティサイトです。

## ■ このサービスへの思い・作りたい理由
**【コンセプト：地元を「推す」ことが、地域の力になる】**

幼い頃から肌で感じてきた「地元の良さ」。それをより多くの人に伝えたいという想いが、このサービスの原点です。

テレビ番組で地域の魅力がランキング形式で熱く語られるシーンを見て、「これこそが地域を盛り上げる鍵だ」と直感しました。お互いの「地元の自慢」をぶつけ合い、切磋琢磨することで、隠れた魅力が次々と掘り起こされていく。そんな活気ある場を、SNSという形で実現したいと考えました。

地域のPRに励む人たちの熱意を、もっと身近に、もっと多くの人に届けたい。ユーザーの皆さんと一緒に、地元の良さを再発見し、楽しみながら日本を元気にしていきたい。一人ひとりの「好き」という気持ちが地域の力に変わっていく、そんな笑顔あふれる未来をこのサービスでカタチにしていきます。

## ■ ユーザー層について
**ターゲットユーザー層**

地元を応援したい人（発信者） 「大好きな地元をもっと知ってほしい！」という想いを持ち、ランキングを上げるために魅力を発信してくれる層。

SNSをよく使う旅行好き（拡散者） 「まだ誰も知らない穴場を見つけたい・広めたい」という、情報感度が高い層。

リアルな情報を探している人（閲覧者） 「ガイドブックにない、地元の人の生の声を聞きたい」という旅行・移住検討層。

**このターゲットにした理由**

「教えたい」と「知りたい」の循環 地元のファンが発信する熱量の高い情報を、情報を探している人に届けるサイクルを作りたいと考えました。

ランキングによる楽しさの提供 ただ投稿するだけでなく、順位が動くワクワク感を取り入れることで、ユーザーが継続して参加したくなる仕組みを目指しました。

技術で地域に貢献したい プログラミングの学習成果として、地元の活性化という社会課題に自分なりのアイデアで挑戦したいという思いから設計しました。

## ■ サービスの利用イメージ

1. あなたの「推し」を写真とコメントで投稿！ 地元のお気に入りの店や、旅先で見つけた隠れた名所を写真と一緒に投稿します。 例えば、「地元で愛される絶品ラーメン！この味をもっと知ってほしい！」といったあなたの発信が、その地域をPRする貴重な一票になります。

2. 「ポイント投票」がランキングに直結！ 【ココじゃ】の最大の特徴は、ユーザーの投稿がそのまま都道府県のランキングポイントに反映される仕組みです。 一般的なSNSのような「いいね」や「コメント」での交流ではなく、「投稿すること＝自分の好きな地域に投票すること」という、シンプルで熱い応援スタイルを楽しめます。

3. ランキングが新しい「旅のきっかけ」に！ みんなの投稿が集まることで、リアルタイムに都道府県の魅力度ランキングが変動します。 「自分の投稿で地元の順位が上がった！」という達成感や、「今この県が盛り上がっているから行ってみようかな？」という新しい発見が生まれ、実際に足を運ぶきっかけを作ります。

4. ひとり一人の「応援」が地域の力になる 特定の観光地だけでなく、ユーザー一人ひとりが主役となって地域の魅力を発掘し、順位として可視化します。このランキングが盛り上がることで、地域の魅力を再発見し、観光客の増加や街の活気へとつなげていくことを目指しています。

## ■ ユーザーの獲得について
このサービスをたくさんの人に知ってもらうために、まずは身近なところから始めて、徐々に広げていく計画です。

**1. まずは「知ってもらう」ためのアクション**

SNSでの地道な発信 サービス専用のアカウントを作り、地域の魅力的なランキング情報を画像や動画で発信します。また、友人や知人に直接使ってもらい、フィードバックをもらいながら広めていきます。

**2. 「見つけてもらう」ための工夫**

直感的に伝わるデザイン 初めてサイトに来た人が「おもしろそう！」と一目で思えるよう、ランキングの見た目や使いやすさ（UI/UX）にこだわります。

**3. これから挑戦したいこと（次のステップ）**

ブログや特集記事の作成 「絶対に行ってほしい！〇〇県の隠れた名店」といった、読み物としても楽しめるコンテンツを増やし、ファンを増やします。

「シェアしたくなる」仕掛け作り ユーザーが自分の好きな地域のランキング結果を、X（Twitter）やInstagramで簡単にシェアできるボタンを設置します。

検索に強いサイト設計（SEO） 「都道府県名 穴場スポット」などで検索したときにヒットするよう、ランキングページを誰でも見られるように公開し、中身を充実させます。

地域とのつながりを作る 将来的には、地域の自治体や観光協会とも協力して、実際のイベント情報と連動した「公式ランキング」などの企画にも挑戦したいです。

## ■ サービスの差別化ポイント・推しポイント

■ サービスの差別化ポイント
1. ユーザーがつくる「リアル」なランキング プロが作ったガイドブックではなく、地元の人や旅行者の「生の視点」がコンテンツの主役です。一人ひとりの何気ない投稿が、まだ見ぬ地域の魅力を掘り起こします。

2. 競い合うから楽しい！ゲーム感覚の応援 都道府県ごとにランキング化されるため、「自分の県を1位にしたい！」という応援する楽しさが生まれます。クラス対抗戦のようなワクワク感で、地域を盛り上げることができます。

3. 「投稿＝一票」のシンプルな仕組み 一般的なSNSのようなコメント機能はなく、**「投稿してポイントを贈る」**というアクションに特化しています。交流の気疲れがなく、誰でも気軽に「応援」に参加できるのが特徴です。

■ このサービスの「推し」ポイント
1. 地域活性化へのダイレクトな貢献 あなたの投稿がそのまま地域のPRになり、観光客やお店のファンを増やすきっかけを作ります。使うだけで「地域貢献」につながるサービスです。

2. エンタメ性バツグンの参加型体験 ただ情報を眺めるだけでなく、自分のアクションでランキングが動く面白さがあります。応援が目に見える形で積み上がる、新しい体験を目指しました。

3. 将来的な「話題性」への期待 「あの地域がいま熱い！」というトレンドがランキングで一目でわかります。テレビやメディアのランキング番組と連動するような、社会にインパクトを与える可能性を秘めています。

## ■ 機能候補

**MVPで必要な機能（詳細）**

1. **ユーザー登録・ログイン機能:**
    - **登録:**
          - メールアドレス、パスワード（確認入力含む）、ユーザー名（ニックネーム）の入力フォーム。
          - 利用規約への同意チェックボックス。
          - プライバシーポリシーへの同意チェックボックス。
          - 登録ボタン。
          - 登録完了メール送信（メールアドレス確認のため）。
    - **ログイン:**
          - メールアドレスまたはユーザー名、パスワードの入力フォーム。
          - ログインボタン。
          - 「パスワードを忘れた場合」リンク（パスワードリセット機能への誘導）。
2. **都道府県選択機能(投稿機能の一部):**
    - 投稿画面で都道府県を選択する。
    - 選択方法：
          - プルダウンリスト（都道府県名一覧）。
    - 選択された都道府県は投稿データに紐付けられる。
3. **投稿機能:**
    - **写真アップロード:**
          - 複数枚の写真を選択してアップロード可能。
          - 画像サイズ(390 × 260)、ファイル形式の制限（例：JPEG、PNG、最大5MB）。← 検討中
          - アップロード中のプレビュー表示。
    - **テキスト投稿:**
          - テキスト入力エリア（文字数制限あり、例：最大500文字）。
          - 改行、簡単な装飾（例：太字、斜体）が可能であれば望ましい。
    - **ハッシュタグ付与:**
          - ハッシュタグ入力フィールド（複数入力可能、カンマ区切りなど）。
          - 入力補完機能（過去に使用されたハッシュタグの候補表示）があると便利。
    - 投稿ボタン。
4. **ランキング表示機能:**
    - 都道府県紹介コンテンツ(47都道府県)
    - データがMVP時は少ないため、将来への期待感を示すことに重点を置きます。
5. **「いいね！」機能:**
    - 投稿に対して「いいね！」ボタンをクリックすることで「いいね！」を付与。
    - 「いいね！」の数は投稿に表示。
    - 同じユーザーが同じ投稿に複数回「いいね！」できないように制御。
6. **コメント機能:**
    - 投稿に対してコメントを投稿できる。
    - コメント入力エリア。
    - コメント投稿ボタン。
    - 投稿されたコメントの一覧表示（投稿日時、ユーザー名、を含む）。
    - コメントの削除機能（投稿者本人と管理者のみ）。
7.  **ユーザープロフィール機能（本人と各ユーザー）:** 
    - ユーザー情報⁨⁩（ユーザー名、ユーザーID、アイコン、自己紹介文）
    - 投稿履歴
    - プロフィール編集(ボタン)（本人のみ編集可能）
8. **検索機能**
    - キーワード入力欄。
    - 検索前は、新着投稿またはおすすめ投稿を表示。
9. **フォロワー・フォロー一覧機能**
    - 本人のプロフィールと各ユーザーのプロフィールで閲覧可能。
10. **設定機能**
    - ログアウト
    - 利用規約
    - プライバシーポリシー
    - お問い合わせ
11. **トップページ**
    - サービス概要
    - ランキング形式の見送り（「注目の都道府県」として数県（例えば5県）をピックアップ）
    - 投稿フォーマットの紹介
    - ログイン/登録へのリンク
    - お問い合わせ
    - 利用規約
    - プライバシーポリシー
12. **新着投稿**
    - アイコン
    - ユーザーID
    - 都道府県名
    - 画像
    - いいね！
    - コメント
    - テキスト（キャプション）、ハッシュタグ

**本リリース後に取り入れてみたい機能**

- **マップ表示機能:** 投稿された場所を地図上に表示する機能。MVPでは必須ではないため除外。Google Maps Platform APIなどの連携が必要となり、開発コストが増加するため。
- **多言語対応:** 複数言語に対応する機能。MVPでは日本語のみでリリースし、ユーザーの反応を見て多言語展開を検討。
- **自治体・企業連携機能:** 自治体や企業が公式情報の発信やキャンペーンなどを実施するための機能。MVPでは個人ユーザーの投稿に焦点を当てるため除外。API開発なども必要になるため、開発コストが増加する。

## ■ 機能の実装方針予定（CRUD以外 - 具体例と技術要素）

1. **マップ表示:**
    - **実装:** Google Maps Platform API (JavaScript API) を使用。投稿データに緯度経度情報を保持し、地図上にマーカー表示。マーカークリックで投稿情報をポップアップ表示。
    - **技術:** Google Maps JavaScript API, ジオコーディング (住所→緯度経度変換)。
2. **検索:**
    - **目的:** 情報の迅速な検索。
    - **実装:** オートコンプリート機能でユーザビリティ向上。
3. **通知:**
    - **目的:** ユーザーエンゲージメントの向上。
    - **実装:** Webプッシュ通知（ブラウザ向け）、アプリ内通知（ネイティブアプリ向け）を実装。「いいね！」「コメント」「フォロー」の通知。通知設定機能も実装。
    - **技術:** Web Push API, 各プラットフォームのプッシュ通知サービス (例: Firebase Cloud Messaging)。
4. **多言語対応:**
    - **目的:** ユーザー層の拡大。
    - **実装:** i18nライブラリ (例: React Intl, i18next) を使用し、言語ごとのテキストリソースを管理。言語切り替え機能実装。必要に応じて機械翻訳API (例: Google Translate API) を活用。
    - **技術:** i18nライブラリ、機械翻訳API。
5. **自治体・企業連携:**
    - **目的:** 公式情報発信と連携強化。
    - **実装:** 自治体・企業向けAPIを提供し、情報連携を容易化。専用の管理画面を用意し、情報投稿・管理を可能にする。地域イベント、観光情報、共同キャンペーン、データ連携などを想定。
    - **技術:** RESTful API設計、管理画面構築。
